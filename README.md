# face_looker

#### 介绍
利用人工智能生成不同视线方向的人脸图像网格，然后用这些图像创建一个能够实时跟随用户光标的交互式人脸。

这个项目分为两部分：
1. python脚本生成百来张朝向不同方向的人脸图片
2. React生成动态网页，让图片目光指向鼠标
#### 准备工作
- 一张 512×512 像素的人脸照片（可以是你的照片或任何肖像照）
- Node.js（用于 React 实现）（安装Node.js到非中文or空格目录下）
- https://replicate.com/account/api-tokens（这个是所需模型的托管平台）



#### 运行mian.py脚本的参数
代码参数
| 参数名           | 示例值            | 说明                                   |
|------------------|-------------------|----------------------------------------|
| `--image`        | `./my_face.jpg`   | 输入图像路径（要处理的那张脸）        |
| `--out`          | `./out`           | 输出目录，生成的图片会放在这里         |
| `--min`          | `-15`             | 眼睛偏移的最小值                       |
| `--max`          | `15`              | 眼睛偏移的最大值                       |
| `--step`         | `3`               | 步长（值越小生成的图片越多）           |
| `--size`         | `256`             | 输出图像大小（最终为 256×256 像素）    |
| `--skip-existing`| *(无值)*          | 跳过已存在的图片，不重新生成           |


- Node.js是一个能让 JavaScript（运行在浏览器上的语言）在电脑上运行的环境
这个语言有3个优点
1. 基于 V8 引擎，速度快得离谱
2. 任务完成后系统会主动通知你，而不是你不断检查它好了没，当任务还没完成时，不会让系统停在那里等结果
对于高并发任务非常高效
Node 执行任务时不会死等
 可以继续处理其他任务
 一旦有输入/输出完成，Node 会回调/事件通知你
3. 全世界最大的软件包生态
react在浏览器上运行，但是依赖的一大坨组件几乎是javascript编写的，所以react依赖于Node.js

- Hook函数= 一套以 use* 开头的函数，允许你在函数组件里使用 React 的特性（状态、生命周期、context 等）。
- React 项目里有三类代码：

| 类别           | 放哪里              | 用途                       |
|----------------|---------------------|----------------------------|
| 组件           | `src/components`    | 负责画东西（UI 展示）      |
| 数据逻辑 / 行为 | `src/hooks`         | 负责算东西（处理逻辑、状态） |
| 工具函数       | `src/utils`         | 跟 React 无关的通用功能     |
